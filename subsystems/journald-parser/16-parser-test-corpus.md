# 16. Parser Test Corpus

## Normative Requirements

- `[JPARSER-CORPUS-001]` The parser test corpus MUST include tests for empty input.
- `[JPARSER-CORPUS-002]` The parser test corpus MUST include tests for a single record.
- `[JPARSER-CORPUS-003]` The parser test corpus MUST include tests for multiple records.
- `[JPARSER-CORPUS-004]` The parser test corpus MUST include tests for missing final newline.
- `[JPARSER-CORPUS-005]` The parser test corpus MUST include tests for duplicate fields.
- `[JPARSER-CORPUS-006]` The parser test corpus MUST include tests for oversized fields.
- `[JPARSER-CORPUS-007]` The parser test corpus MUST include tests for oversized records.
- `[JPARSER-CORPUS-008]` The parser test corpus MUST include tests for malformed binary fields.
- `[JPARSER-CORPUS-009]` The parser test corpus MUST include tests for truncated binary payloads.
- `[JPARSER-CORPUS-010]` The parser test corpus MUST include tests for invalid field names.
- `[JPARSER-CORPUS-011]` The parser test corpus MUST include tests for non-UTF-8 message bytes.

Required tests:

-   empty input
-   single record
-   multiple records
-   missing final newline
-   duplicate fields
-   oversized field
-   oversized record
-   malformed binary field
-   truncated binary payload
-   invalid field name
-   non‑UTF‑8 message bytes
