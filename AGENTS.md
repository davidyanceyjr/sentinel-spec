# Agent Instructions

## Scope
This repository contains the canonical production specification for the Sentinel Linux terminal application.

## Core rules
- Treat the specification as normative unless explicitly marked exploratory.
- Do not invent requirements, features, or implementation details.
- If something is missing, mark the gap instead of filling it.
- Preserve meaning unless explicitly asked to change meaning.
- Keep edits scoped to the requested files or sections.

## Review mode
When asked to review:
- Identify contradictions.
- Identify unverifiable or ambiguous requirements.
- Identify missing production concerns.
- Do not rewrite during review unless asked.

## Rewrite mode
When asked to rewrite:
- Use normative language: MUST, MUST NOT, SHOULD, MAY.
- Make requirements testable and unambiguous.
- Preserve original meaning.
- Do not broaden scope.

## Refactor mode
When asked to extract or reorganize:
- Move content without changing meaning.
- Preserve ordering where practical.
- Leave references intact.
- Do not drop content silently.

## Output expectations
- Summarize intended changes before editing.
- Flag semantic changes explicitly.
- Prefer small reviewable diffs.
